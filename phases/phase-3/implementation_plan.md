# Phase 3 Plan: Comparison Engine — Percentiles, Peers, and Flags

## Context

Phase 2 is complete and approved. MongoDB Atlas has 2,532 WA school documents loaded and verified (19/19 checks passed). The pipeline uses `data/schools_pipeline.json` as the intermediate store, with numbered scripts processing it sequentially.

Phase 3 adds the computational layer that makes comparisons meaningful: derived ratios, peer group assignments, percentile rankings, demographic-adjusted performance regression, and structured climate/equity flags with metadata.

---

## Critical Data Gaps

Two fields mentioned in the foundation doc are **not available** in our CCD extract:
- **LOCALE** (city/suburb/town/rural) — column not present in `data/ccd_wa_directory.csv`
- **Title I status** — column not present in CCD

**Mitigation:** Peer groups use **FRL band + enrollment band + school level** instead. FRL is a strong poverty proxy (the same signal Title I captures). School level (Elementary/Middle/High) partially captures the structural differences locale would provide.

---

## Architecture Decision: Per-Level Regression

**Verified empirically:** A single statewide regression gives Fairhaven a z-score of 0.89 (YELLOW — would fail the gate check). A per-level regression (within Middle schools only) gives Fairhaven z-score 1.33 (GREEN).

This is educationally defensible: elementary, middle, and high schools have different proficiency-FRL curves (elementary R²=0.72, middle R²=0.69, high R²=0.56). Comparing a middle school against elementary-dominated statewide data distorts the comparison.

**Distribution for Middle schools at ±1.0 SD:** 12.3% green, 78.4% yellow, 9.3% red — close to the 15/70/15 target.

**Note:** This is a deviation from the build sequence, which specifies a single statewide regression. Documented in `phases/phase-3/decision_log.md`.

## Script-by-Script Breakdown

### 11_load_crdc_enrollment.py — CRDC Enrollment by Race

```
PURPOSE: Load race-specific enrollment counts from CRDC enrollment.csv.
         Phase 2 used this file only to set crdc_combokey — it did NOT
         extract the race breakdown columns. Phase 3 needs them as
         denominators for discipline disparity ratio computation.
INPUTS:  data/crdc_wa/enrollment.csv, data/schools_pipeline.json
OUTPUTS: Adds enrollment.crdc_by_race to each school
JOIN KEYS: COMBOKEY → _id (direct match)
```

**Logic:**
1. Read `enrollment.csv`, extract `SCH_ENR_{race}_M` and `SCH_ENR_{race}_F` columns
2. Sum M+F for each of the 7 races → `{"white": 251, "hispanic": 81, "black": 7, ...}`
3. Store as `enrollment.crdc_by_race` and `enrollment.crdc_total`
4. Schools without CRDC match get no `crdc_by_race` field

**Fairhaven expected:** white=251+142=393? (Need to verify M+F sums during implementation)

### 12_compute_ratios.py — Derived Ratios

```
PURPOSE: Compute all derived ratios from base data fields.
INPUTS:  data/schools_pipeline.json
OUTPUTS: Adds/overwrites derived section for each school
```

**Computations:**

| Ratio | Formula | Null when |
|-------|---------|-----------|
| `derived.student_teacher_ratio` | enrollment.total / staffing.teacher_fte_total | teacher FTE null or 0 |
| `derived.counselor_student_ratio` | enrollment.total / staffing.counselor_fte | counselor FTE null or 0; also set `derived.no_counselor: true` |
| `derived.chronic_absenteeism_pct` | 1.0 - academics.attendance.regular_attendance_pct | attendance null |
| `derived.proficiency_composite` | (ELA + Math) / 2 | either proficiency null |
| `derived.discipline_disparity` | per-race dict of (suspension_rate_R / suspension_rate_white) | no CRDC, no white baseline, race enrollment < 10 |
| `derived.discipline_disparity_max` | highest ratio across all races | same |
| `derived.discipline_disparity_max_race` | which race has highest ratio | same |

**Discipline disparity detail:**
- Suspension numerator per race = ISS + OSS_single + OSS_multiple (non-IDEA, from `discipline.crdc`)
- Suspension denominator per race = `enrollment.crdc_by_race[race]`
- Only compute for races with 10+ enrolled students (below 10, single incidents create misleading ratios)
- If white enrollment = 0 or white suspension rate = 0, cannot compute ratio → null + `"no_white_baseline": true`

**Fairhaven expected:**
- Student-teacher ratio: 588 / 33.6 ≈ 17.5
- Counselor ratio: 588 / 2.0 = 294
- Chronic absenteeism: 1.0 - 0.6003 = 0.3997
- Proficiency composite: (0.649 + 0.549) / 2 = 0.599

### 14_compute_percentiles.py — Percentile Rankings

```
PURPOSE: Calculate percentile ranks (0-100) for 8 curated metrics within
         district, statewide, and peer cohort.
INPUTS:  data/schools_pipeline.json, flag_thresholds.yaml
OUTPUTS: Adds derived.percentiles section
```

**8 curated metrics** (from `flag_thresholds.yaml`):

| Metric | Field | Polarity |
|--------|-------|----------|
| ELA proficiency | academics.assessment.ela_proficiency_pct | higher = better |
| Math proficiency | academics.assessment.math_proficiency_pct | higher = better |
| Regular attendance | academics.attendance.regular_attendance_pct | higher = better |
| Student-teacher ratio | derived.student_teacher_ratio | lower = better |
| Counselor-student ratio | derived.counselor_student_ratio | lower = better |
| Per-pupil expenditure | finance.per_pupil_total | higher = better |
| Chronic absenteeism | derived.chronic_absenteeism_pct | lower = better |
| Discipline rate | discipline.ospi.rate | lower = better |

**Percentile formula:** `pct = (count_below + 0.5 * count_equal) / total * 100`, rounded to integer. For "lower is better" metrics, flip: `pct = 100 - pct`.

**Scope rules:**
- **State:** All WA schools with non-null value for that metric
- **District:** Within same `district.nces_id`, only if 2+ schools have data (81 single-school districts → null)
- **Peer:** Within same `derived.peer_cohort`, only if 5+ schools have data

**Storage:**
```json
"derived": {
  "percentiles": {
    "ela_proficiency_pct": {"state": 72, "district": 85, "peer": 91},
    "chronic_absenteeism_pct": {"state": 15, "district": null, "peer": 22}
  }
}
```

### 15_regression_and_flags.py — Performance Regression + All Flags

```
PURPOSE: Run FRL-vs-proficiency regression per school level, compute
         residuals and performance flags. Apply climate/equity flag
         thresholds with structured metadata. Compute flag_absent_reason
         for every school missing a flag.
INPUTS:  data/schools_pipeline.json, flag_thresholds.yaml
OUTPUTS: Adds derived.performance_flag, derived.regression_*, derived.flags
```

**Part A: Performance Regression**

1. Collect schools with non-null FRL + ELA + Math proficiency (1,249 schools)
2. Group by `derived.level_group` (Elementary ~773, Middle ~204, High ~200, Other ~72)
3. For groups with 30+ schools: fit `LinearRegression(X=frl, y=composite)` per group
4. For groups with <30: use statewide all-levels regression as fallback
5. Compute residual and z-score per school
6. Flag assignment at ±1.0 SD:
   - z > +1.0: `"outperforming"` (green)
   - z < -1.0: `"underperforming"` (red)
   - else: `"as_expected"` (yellow)

**Stored per school:**
- `derived.performance_flag`: green/yellow/red string
- `derived.regression_predicted`: predicted composite from model
- `derived.regression_residual`: actual - predicted
- `derived.regression_zscore`: residual / SD
- `derived.regression_group`: which level group was used
- `derived.regression_r_squared`: R² of the group's model

**Fairhaven verification:** Middle group, z=1.33, flag=outperforming

**Part B: Climate & Equity Flags**

| Flag | Metric | Yellow | Red | Fairhaven Expected |
|------|--------|--------|-----|-------------------|
| Discipline disparity | derived.discipline_disparity_max | >2.0 | >3.0 | TBD (verify during impl) |
| Counselor ratio | derived.counselor_student_ratio | >400 | >500 | green (294) |
| Chronic absenteeism | derived.chronic_absenteeism_pct | >0.20 | >0.30 | red (0.40) |

**Flag metadata structure** (same text for every school with same flag):
```json
{
  "color": "red",
  "raw_value": 0.3997,
  "threshold": 0.30,
  "threshold_source": "USED chronic absenteeism benchmarks; foundation document v0.2",
  "what_it_means": "Nearly 40% of students are chronically absent...",
  "what_it_might_not_mean": "May reflect community factors, not school climate...",
  "parent_question": "What is the school doing to address chronic absenteeism?"
}
```

**Part C: Flag Absent Reasons**

For every school missing a flag, compute `flag_absent_reason` with high-confidence categories only:

| Reason | When to apply |
|--------|---------------|
| `grade_span_not_tested` | School's grade span doesn't include tested grades (3-8, 10). Check CCD `level` and grade range fields. |
| `suppressed_n_lt_10` | The underlying metric has a suppression marker (OSPI N<10 or CRDC -5 small_count). |
| `data_not_available` | Everything else — no inferences beyond what the data directly supports. |

**Storage per flag:**
```json
{
  "color": null,
  "flag_absent_reason": "grade_span_not_tested"
}
```

**No speculation:** If a school is missing data and we can't confidently determine why, use `data_not_available`. The three categories are exhaustive — no additional reasons invented during implementation.

**Missing data handling (flags that ARE null):**
- Schools missing CRDC: discipline disparity flag = null + `flag_absent_reason: "data_not_available"`
- Schools missing attendance: absenteeism flag = null + appropriate `flag_absent_reason`
- Schools missing staffing: counselor flag = null + appropriate `flag_absent_reason`
- Performance flag null: schools missing FRL or both proficiency scores → appropriate `flag_absent_reason`

**Sanity schools** (5-10, identified by querying data during implementation):
1. Low-FRL school with high scores → should be "as_expected" (not green — that's demographics working as predicted)
2. High-FRL school with low scores → should be "as_expected" or "underperforming"
3. School with high discipline disparity → yellow or red discipline flag
4. Single-school district → null district percentiles
5. School missing CRDC → null disparity flags with data_unavailable annotation
6. Low-FRL school with mediocre scores → "underperforming" (red — the tool's key insight)

## flag_thresholds.yaml Structure

Located at project root alongside `cleaning_rules.yaml`. Contains all thresholds, band definitions, percentile metric configs, and flag metadata templates. Separate from cleaning_rules.yaml because this is about interpretation, not data ingestion.

Key sections: `enrollment_bands`, `frl_bands`, `level_groups`, `regression` (threshold_sd, min_group_size, method), `percentile_metrics` (8 metrics with field paths and polarity), `flags` (3 flags with thresholds, sources, and metadata text templates).
